# Roadmap do Projeto

## Fase 1 — Protótipo estático

A versão atual valida os fluxos principais de cadastros, romaneios, impressão e relatórios usando apenas HTML, CSS, JavaScript e armazenamento local do navegador.

## Fase 2 — MVP com backend

A próxima etapa recomendada é implementar autenticação real, banco de dados e API para compartilhamento dos dados entre usuários autorizados.

| Componente | Sugestão |
|---|---|
| Frontend | React + Vite ou evolução gradual do HTML atual. |
| Backend | Node.js + Express, Next.js ou Supabase. |
| Banco | PostgreSQL, Supabase ou Neon. |
| Hospedagem | Vercel, Netlify, Render, Railway ou Supabase. |

## Fase 3 — Operação completa

A plataforma pode evoluir com permissões por perfil, exportação de relatórios, integração com planilhas, geração avançada de PDF, auditoria e backup automático.

## Fase 4 — Integrações

Integrações futuras podem incluir WhatsApp Business, assinatura digital, importação de dados fiscais e assistente de IA para preenchimento automático de romaneios.
